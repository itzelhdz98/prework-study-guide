HTML


Definitions:

  • Hypertext Markup Language (HTML) is composed of elements that give webpages structure.
  • HTML elements contain content within an opening tag, < element name >, and a closing tag, < / element name >.
  • HTML elements can contain attributes enclosed within the opening tag.
  • Child elements are elements nested inside other elements.
  • Parent elements are elements that hold all the child elements.

Elements:

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !DOCTYPE element declares what type of document to expect when opened.
  • Title element defines the title of the webpages and is nested in the head.
  • Style element defines CSS styles and is nested in the head.
  • Link element connects external resource to the HTML document and is nested in the head.
  • Meta element defines metadata and is nested in the head.
  • Script element embeds client-side scripts in the HTML document and is nested in the end of the body.
  • Heading elements are denoted by h1, h2, h3, h4, h5, and h6. They represent the level of heading a text block represents.
  • The paragraph element, denoted by p, represents a paragraph or block of text.
  • Unordered lists element is denoted by ul.
  • Ordered lists element is denoted by ol.
  • Each list item is denoted by li and is nested in either ul or ol.
  • Image elements are denoted by img.
  • To create a line break between two blocks of content, use the br element.
  • An anchor element, denoted by a, creates links to a webpage. It includes the href attribute to point to the URL.

CSS


Types of CSS:

  • Cascading Style Sheets (CSS) is a programming language that defines the style of webpages.
  • Inline CSS is styling an element directly on the HTML file.
  • Internal Style Sheet has CSS rules in the head section of an HTML file.
  • External CSS Style Sheet is a file linked to the head element of an HTML file.
  • Separation of concerns means separating code into different files based on use.

CSS Rules

  • A CSS rule is composed of a selector and declaration(s).
  • A selector defines the element to which the declarations will apply.
  • A declaration in CSS is composed of properties and their values.
  • The * selector applies declarations to all visible elements.
  • A margin is a property that indicates how much space we want around the outside of an element.
  • A padding is a property that indicates how much space we want around the content inside an element.

Git


Definitions:

  • Git is a version control system.
  • GitHub is a web interface for using Git.
  • Repository (repo) is a folder in which all project's files and their version histories are stored.
  • Branch is a version of the repo that allows work to be done without affecting other branches.
  • Fork is a new repository that from a parent repo.
  • Markdown (.md) is a way to write content from plain to formatted text.
  • Pull request (PR) is a request for changes made to a branch to be pulled into another branch.
  • Merge is when after a PR is approved, the commit will be pulled in from one branch to another.
  • Issues permit users to report issues or bugs and track the progress of assigning the fix.
  • Federalist is a platform that deploys a website from a GitHub repository.

Developing a Website on GitHub:

  1. Issue will be opened via the project board
  2. A new branch will be created from the most recent version of the repo.
  3. Commits will be added to their respective branches.
  4. Pull requests will be opened for other team members to review changes and discuss commits.
  5. Federalist is built. Reviewers will approve or comment on the change. Once the PR is approved, the commits merge into the main branch and are published on the live site.

Git Commands:

  • Git commands begin with git.
  • git status: checks which branch we are currently working on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main: switches to main branch.
  • git add -A: adds all modifications in the current working branch.
  • git commit -m "message": captures a snapshot of the currently staged changes.
  • git pull origin main: pulls from the main branch in the repo.
  • git push origin branch-name: pushes the changes in the local branch to the remote branch.

JavaScript


Definitions:

  • JavaScript (JS) is a programming language that can add interactivity to a website.
  • A variable is a named container that allows us to store data in our code.
  • Declaring a variable: var variable-name;
  • Control flow is the order in which a computer executes code in a script.
  • JS code runs from the first statement at the top of the page.

Data Types:

  • String: A sequence of text. Enclosed within ' ' or " ".
  • Number: Numbers. Not enclosed within quotes.
  • Boolean: True or False. Not enclosed within quotes.
  • Array: Stores multiple values in a single reference.
    • Enclosed within [ , , ].
    • A single value in the array is called an index.
    • variable-name[0],variable-name[1],...

Operators:

  • Addition (+): Adds two numbers or combines two strings together.
  • Subtration (-)
  • Multiplication (*)
  • Division (/)
  • Assignment (=)
  • String Equality (===): Tests if two values are equal and of the same data type. Returns Boolean value.
  • Not (!): Returns the logically opposite value of what it precedes.
  • Does-Not-Equal (!==): Tests whether two values are not equal. Returns Boolean value.

Conditionals:

  • Conditionals are code structures used to test if an expression returns true or not.
  • Commonly used in the form of if...else statements.
  • In JS, if statements will check if a statement is truthy.If the condition is truthy, it wiill do what is within the curly brackets.
  • If condition isn't true/truthy, then the computer will ignore whatever is in between the curly brackets.
  • Falsy values: 0, -0, on, " ", null, undefined, NaN, false(Boolean).
  • If a value is not falsy, it is considered truthy.

Functions:

  • Functions are a way of packaging functionality that wish to reuse.
  • There are some built-in functions in the system and there are defined functions.
  • It is important to not only declare the function, but to call the function to use it.
  • function function-name(argument){
    //code block}
  • console.log function: output a message to web console by adding an argument.
  • math.random function: selects a random number.

Events

  • Code structures that listen for activity in the browser and run code in response.

For Loop:

  • Performs code within curly brackets as long as conditions are met.
  • Conditions: Starting point, condition, iterations.
  • for (var x=0; x < variable.length; x++) {
    //code block}
